# Moonlab v0.2.0 -- "Quantum source, quantum safe"

This is the first release in the v0.x arc to unify a full-stack
quantum simulator with a full-stack post-quantum cryptography
foundation in one library, driven by the same Bell-verified quantum
RNG.  Two themes: **make the quantum capabilities credible**
(tensor-network topology, native autograd, error mitigation,
expanded measurement + Bell surfaces) and **make the cryptography
story real** (FIPS 202 SHA-3 / SHAKE + FIPS 203 ML-KEM at all three
parameter sets, seeded by moonlab_qrng_bytes).

## Headline features

### End-to-end quantum -> quantum-safe KEM

The same Bell-verified QRNG that ships as `moonlab_qrng_bytes`
(stable since v0.1.2) now feeds directly into NIST-standard post-
quantum key encapsulation:

```c
#include <moonlab/moonlab_export.h>

uint8_t ek[MOONLAB_MLKEM768_PUBLICKEYBYTES];
uint8_t dk[MOONLAB_MLKEM768_SECRETKEYBYTES];
uint8_t ct[MOONLAB_MLKEM768_CIPHERTEXTBYTES];
uint8_t K_alice[32], K_bob[32];

moonlab_mlkem768_keygen_qrng(ek, dk);
moonlab_mlkem768_encaps_qrng(ct, K_bob, ek);
moonlab_mlkem768_decaps(K_alice, ct, dk);
/* K_alice == K_bob */
```

Three NIST parameter sets are shipped (ML-KEM-512 / 768 / 1024),
each exported on the stable ABI and bound in Python via
`moonlab.crypto.mlkem`.

### Native reverse-mode autograd, now wired into VQE

Adjoint-method gradients for parameterised circuits, including
controlled rotations (CRX / CRY / CRZ).  `vqe_compute_gradient`
automatically uses adjoint for hardware-efficient ansatz in
noise-free simulation, falling back to parameter-shift otherwise.
Measured PSR/adjoint speedup on a 6-qubit HEA grows 1.0x -> 1.9x
-> 3.5x -> 6.7x as parameter count goes 12 -> 96, linear in
parameter count as theory requires.

### Error mitigation as a first-class subsystem

New `src/mitigation/` with zero-noise extrapolation (linear /
Richardson / exponential estimators) and probabilistic error
cancellation primitives (one-norm cost, signed sampling,
aggregation).  Recovered noiseless `<Z>` from an 8%-depolarised
sample to 1e-13 in the integration test.

### Real-space topology at O(10^8)-site scale

The MPO Chebyshev-KPM path is closed end-to-end on the QWZ Chern
insulator.  The Bianco-Resta local Chern marker computed through
the full tensor-network pipeline matches the dense Schulz-iteration
reference to machine precision (|diff| = 0.0000 on L=4 lattices).

### Measurement and nonlocality completeness

- POVM measurement with Kraus-completeness verification.
- Weak Z-measurement with tunable strength (0 = non-disturbing,
  1 = projective).
- Quantum mutual information I(A:B) over arbitrary disjoint
  partitions.
- Mermin polynomial on |GHZ_3> (classical bound 2, quantum max 4).
- Mermin-Klyshko M_N on |GHZ_N> (normalised classical bound 1,
  quantum max 2^((N-1)/2)); matches theory to 1e-9 for N = 2..5.

### Device-independent QRNG primitives

- Pironio-style certified min-entropy H_min(S) per measurement bit.
- Toeplitz-hash extractor (2-universal strong extractor with
  leftover-hash-lemma bound).
- `qrng_di_raw_bytes_for_output` sizing helper.

### Extended noise surface

Composite (sequential) channels, convex-mixture channels, and a
16-probability correlated two-qubit Pauli channel that can model
XX-biased CNOT errors product-of-channels can't express.

## Compatibility

- **API**: all v0.1.x public symbols stable.  `moonlab_export.h`
  grows to include ML-KEM entry points, constants, and sizes.
- **ABI**: `MOONLAB_ABI_VERSION_MINOR` bumps to 2.
- **Downstream**: QGTL (pinned at v0.1.2) can migrate to v0.2.0
  to pick up the native autograd and the QRNG helpers; no changes
  are required for consumers staying on `moonlab_qrng_bytes`.

## Caveats (read before deploying)

- **ML-KEM validation**: self-regression KAT only, not NIST FIPS 203
  KAT.  The algebra is byte-stable against the reference shipped in
  this tree; full NIST conformance (AES-256-CTR DRBG seed expansion
  + official .rsp comparison) is tracked for v0.2.1.
- **Crypto security posture**: reference implementation.
  Constant-time on non-exotic CPUs, not FIPS-140-certified, not
  hardened for adversarial side-channel environments.  For
  FIPS-certified production crypto, integrate with BoringSSL /
  OpenSSL EVP; the QRNG seed path still applies.
- **MPI scaling**: 2- and 4-node Docker CI is green; no
  cross-host scaling benchmarks published.
- **GPU backends**: Metal is the only backend covered by tests;
  CUDA / OpenCL / Vulkan / cuQuantum compile behind their feature
  flags but are not CI-exercised on production hardware.
- **`hermitian_eigen_decomposition` on complex-Hermitian inputs**:
  returns correct eigenvalues but not eigenvectors; the complex-
  Jacobi rewrite is tracked for v0.3.

## Upgrade notes from v0.1.x

- v0.1.x BELL_VERIFIED QRNG mode was structurally vacuous due to a
  bug in `bell_test_chsh` that reset the input state.  Upgrade to
  v0.2.0 to get real Bell-verification semantics.
- Python `moonlab.algorithms` (VQE / QAOA / Grover / BellTest) now
  runs end-to-end; previously skipped algorithm tests are live.

## What's next (v0.3)

- NIST FIPS 203 official KAT conformance for ML-KEM.
- ML-DSA (FIPS 204) signatures + SLH-DSA (FIPS 205).
- Matrix-product density operator (MPDO) noise simulator.
