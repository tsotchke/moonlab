# MoonLab v1.2.0 — bounded distributed scale and release hardening

Prepared 2026-07-18 and hardened through 2026-07-23; publication is gated on
the verification contract below. Version 1.2.0 turns the post-1.1 CUDA work into a
commit-bound distributed release: sharded state vectors can execute beyond 32
qubits without allocating full remote-shard halos, the public and binding ABI
survives hidden-visibility builds, and every release claim is backed by the ICC
runtime-evidence gate.

## Distributed CUDA and MPI

- Remote one- and two-qubit gates, CNOT, and distributed X/Y expectation values
  exchange bounded chunks. Communication memory no longer scales as two full
  remote partitions per rank.
- The fleet gate runs an N=12 preflight and then the exact N=33 proof with four
  MPI ranks across two distinct hosts, two ranks per host. The trace records the
  clean source SHA, requested and observed topology, GPU endpoint count, and
  halo-exchange count.
- New four-rank regression cases cover remote Hadamard, CNOT, one- and
  two-partition two-qubit gates, and distributed Pauli expectations. They also
  fixed X double-counting and the Y orientation.
- CPU-only MPI builds now provide the weak CUDA device-selection fallback
  required to link on Mach-O as well as ELF.

## Stable ABI 0.6.0

- The binding-consumed native surface is explicitly exported under hidden
  visibility, including dense state/gate/measurement, autograd, algorithms,
  tensor-network, GPU, and distributed families.
- `moonlab_ca_mps_conjugate_pauli` and seven QGT topology one-shots join the
  stable surface.
- `moonlab_complex_double` keeps the public header parseable under MSVC while
  preserving the C99 complex ABI for C consumers.
- Python, Rust, JavaScript/WASM, examples, and native ABI consumers are tested
  against the hidden-visibility shared library. A fresh ICC structural index
  reports zero unimplemented public declarations.

## Numerical, concurrency, and control-plane hardening

- The no-LAPACK SVD fallback again produces an isometry, including the Windows
  and minimal-dependency configurations that exercise it.
- ThreadSanitizer and adversarial concurrency lanes cover the repaired lazy
  initialization, setter, audit, control-plane, and entropy-pool synchronization
  paths. The entropy-pool background-state race -- surfaced by clang-19
  ThreadSanitizer where other schedules missed it -- is closed by publishing the
  running flags under the same lock the stats reader already holds.
- The control plane rejects oversized circuit frames before resource-amplifying
  allocations, and the bounded circuit deserializer rejects ambiguous input and
  no longer loops forever on a payload containing an interior NUL byte.
- Cached measurement validates the POVM structure before use, and the ELF shared
  library is linked resident across a consumer `dlclose` so unloading cannot tear
  down unload-unsafe runtime state.
- The Windows/MinGW build is clean under `-Werror`: the compat shims silence
  `-Wpedantic` on their intentional `#include_next`, and the audit-buffer yield
  includes `<windows.h>` for `SwitchToThread`.
- Large-n tensor-network and TDVP differential checks enforce every case with
  an empty known-divergence quarantine.
- ASan/UBSan, numerical edge, uninitialized-state, scaling, and thread-safety
  traces are relayed into one failure-free release oracle.

## Language packages and release automation

- `@moonlab/quantum-core` reaches parity with the v1.2 surfaces: UCCSD and
  QNG VQE ergonomics with optimizer hyperparameters, the corrected
  random-draw noise-channel surface plus `DeviceNoiseModel` and readout
  error, the Pauli-frame batch shot/detector sampler, and the union-find
  decoder, each with numeric integration coverage (UCCSD H2/LiH at chemical
  accuracy, GHZ shot correlations, channel rates at 4-sigma tolerance, the
  C decoder test vectors, and a seeded sampler-to-decoder round trip).
  `docs/PARITY_MATRIX.md` is re-audited against the actual binding state of
  all four languages, including the surfaces that remain deliberately
  unbound (MPI in Rust/JS, chemistry and error mitigation outside Python).
- Package metadata and runtime version constants are synchronized at 1.2.0 for
  Python, Rust, npm, Debian, and container builds.
- Rust path dependencies carry the same release constraint as their local
  crates, and JavaScript version-contract tests are stamped by the canonical
  version tool.
- Release preflight validates the tag against `VERSION.txt` before registry or
  artifact publication begins.
- crates.io publication uploads the exact certified `.crate` bytes from the
  promoted manifest rather than letting `cargo publish` repackage, so the
  published artifact is byte-identical to the tested one; the workflow fails
  closed if the digest does not match.
- The release matrix includes relocatable Windows x64 and ARM64 SDK archives in
  addition to Linux/macOS libraries and language packages.

## Verification contract

The v1.2.0 tag is cut only from a revision for which all of the following are
green:

- ordinary and hidden-visibility fresh builds;
- the full eligible CTest suite and language-binding suites;
- ASan/UBSan, TSan, numerical, and scaling deep-hunt lanes;
- exact N=33 two-host CUDA/MPI fleet evidence bound to the tagged SHA;
- the five-node MoonLab mesh smoke; and
- the complete GitHub Actions matrix, including Windows x64 and ARM64.

The final machine-readable release certificate must bind these results to one
clean source state. It records the version, commit SHA, Git tree SHA, ICC source
fingerprint, toolchain/profile identifiers, per-lane test counts, evidence and
package SHA-256 digests, MPI/mesh topology and node results, hosted-CI run IDs
and head SHA, and the annotated tag object's target. Bare `PASS` events or
evidence generated before a history rewrite cannot certify the tag.

## Compatibility

MoonLab 1.2.0 remains in the 1.x package compatibility line. Existing 1.1 C
consumers continue to use the same installed namespaces and can feature-gate
new stable entries with `moonlab_abi_version()`. Rebuild static consumers and
language wheels/crates against the 1.2 package metadata before deployment.
