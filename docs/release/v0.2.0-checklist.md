### Known limitations documented

- [x] Multi-node (>1 host) MPI scaling untested.
- [x] CUDA / cuQuantum backends compile-check only — no CI runner
      with NVIDIA hardware.
- [x] WebGPU default ctest coverage requires `-DQSIM_BUILD_JS_DIST=ON`.
- [x] Windows unsupported.
- [x] QTCI for arbitrary (non-monomial) position functions is
      future work; linear-in-coordinate QTCI is the existing
      diagonal-sum MPO.
- [x] `unit_qgt` Haldane Chern-number test is Release-only: the
      Debug build computes the 48x48 Fukui-Hatsugai-Suzuki
      integral with enough float noise that the rounded Chern
      number drifts from ±1 to 0.  Excluded from the ASAN tier.
      Release builds are unaffected.

## Tag preparation (when ready)

When the user calls the go:

1. `git tag -a v0.2.0 -m "..."` on current master.
2. Push `master` + `v0.2.0` to `origin` and `gitlab`.
3. Bump `VERSION.txt` to `0.2.1-dev` on master in a post-tag commit.
4. Draft GitHub Release notes from the CHANGELOG section.

  access to the new `quantum_entropy_ctx_create_hw` / `_destroy`
  helpers and the native autograd.
- Announce the Bell-test bugfix loudly — anyone using the
  BELL_VERIFIED QRNG mode on 0.1.x needs to update to get real
