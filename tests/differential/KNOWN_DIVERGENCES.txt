# KNOWN_DIVERGENCES.txt
#
# Cases the cross-backend differential currently catches diverging because a
# sibling lane is mid-fix. Each line quarantines one (case, backend) pair: for
# the tn_mps backend a quarantine means the driver SKIPS that case's tn leg
# entirely (see below) and counts it as quarantined -- never as a pass, never as
# a failure. Tolerances are NEVER loosened to make HEAD pass; this file is the
# only escape hatch.
#
# Format (whitespace-delimited; '#' comments and blank lines ignored):
#   <case_id>  <backend>  <owning_lane>  <free-form note incl. seed>
#
# case_id/seed reproduce byte-identically from the default corpus
#   scripts/gen_diff_corpus.py --out-dir DIR        (seed 0xB16B00B5)
# so any entry replays in isolation, at any profile (a case is identical whether
# generated in the quick subset or the full matrix).
#
# ---------------------------------------------------------------------------
# LANE: tn-lane  (plain tn_mps backend, src/algorithms/tensor_network)
# ---------------------------------------------------------------------------
# Root cause -- the tn_mps 2-qubit gate path, two coupled failure modes:
#
#   (1) Reversed-CNOT transpose. tn_apply_cnot(control, target) with
#       control > target applies the transposed 2-qubit tensor. Minimal repro:
#       on 3 qubits, H(0) then CNOT(1,0) puts the amplitude on the wrong basis
#       state; dense/reference are exact and disagree with tn_mps.
#   (2) Toffoli (ccx) decomposition expands into a CNOT/T network containing
#       reversed CNOTs, so it inherits (1).
#
# WHY THE WHOLE LEG IS SKIPPED (not just its result compared): the buggy 2q
# path also writes out of bounds. Running it corrupts heap state and poisons
# LATER cases -- the set of secondarily-affected cases shifts run-to-run with
# ASLR (deterministic within a process, not across). Executing the path at all
# is therefore unsafe, so the driver refuses to run the tn leg for any case
# listed here and the rest of the run stays deterministic.
#
# SCOPE: every case in the default full matrix whose circuit contains a reversed
# CNOT (control>target) or a Toffoli is listed -- i.e. every case that can touch
# the corrupting path, not merely the ones observed to shift on a given run.
# Forward-only circuits (rot_cnot_ladder, param_layer, ghz, qft, and the
# forward-CNOT clifford/random cases) reproduce the reference exactly on tn_mps,
# so the tn_mps leg is a live guard everywhere except these entries.
#
# Backends that disagree: tn_mps vs {dense, reference}.  Owning lane: tn-lane.
#

# LANE: tn-lane  (SECOND, independent bug -- handled by an n-cap, not per-case)
# ---------------------------------------------------------------------------
# tn_mps also diverges from the reference on FORWARD-ONLY deep circuits at
# n >= 10 (e.g. param_layer / rot_cnot_ladder at n=10: ry/rz + forward cz or
# forward cx only, NO reversed CNOT, NO Toffoli). The divergence is:
#   * deterministic and reproducible in complete isolation (single-case corpus),
#   * identical under -O0 and -O2 (not a memory-UB artifact),
#   * independent of bond dimension (chi = 32, 64, 256 give the same error),
# so it is a genuine algorithmic tn_mps correctness bug, distinct from the
# reversed-CNOT transpose above. Repro:
#   scripts/gen_diff_corpus.py --out-dir /tmp/c --qubits 10 --depths 8
#   diff_backends /tmp/c/corpus.txt   # param_layer_n10_d08 diverges ~3e-2
#
# tn_mps reproduces the reference EXACTLY for every family and depth at n <= 8,
# so run_cross_diff.sh caps the exact tn_mps oracle at --tn-max-n 8. Cases at
# n >= 10 are recorded as tn-skipped (never as pass), and their correctness is
# still anchored absolutely by dense/python/rust/js vs the numpy reference.
# Raising the cap (--tn-max-n 10/12) re-arms the tn leg there and the harness
# will fail on these divergences -- intended, until the tn-lane fix lands.
